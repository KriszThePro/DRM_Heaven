# IQ_Transmitter (IQ Decoder CLI)

`IQ_Transmitter` is a command-line decoder for IQ files generated by the `MDI_to_IQ` project's QPSK mapping stage.

## What it does

- reads interleaved IQ input (`f32` or `s16`),
- performs nearest-point QPSK demapping,
- recovers payload bits,
- packs bits into bytes (MSB-first),
- writes recovered binary output,
- optionally streams IQ to FL2K through an external command.

## Usage

```powershell
.\x64\Debug\IQ_Transmitter.exe <input.iq> <output.bin> [--format f32|s16] [--report <report.txt>]
.\x64\Debug\IQ_Transmitter.exe <input.iq> --tx-fl2k --fl2k-cmd "<command reading raw u8 IQ on stdin>" [--format f32|s16]
.\x64\Debug\IQ_Transmitter.exe --self-test
```

## Decode Example

```powershell
.\x64\Debug\IQ_Transmitter.exe .\sample_refactor.iq .\sample_refactor.decoded.bin --format f32 --report .\sample_refactor.decode_report.txt
```

## FL2K Transmission Example

The app converts normalized IQ samples (`[-1, 1]`) to interleaved unsigned 8-bit samples and streams them to the command provided by `--fl2k-cmd` via stdin.

```powershell
.\x64\Debug\IQ_Transmitter.exe .\mf_from_pcap.iq --tx-fl2k --format f32 --fl2k-cmd "fl2k_file -s 12000000 -f u8 -c 2 -i -"
```

Use the exact `fl2k_file` arguments required by your local osmo-fl2k build.
