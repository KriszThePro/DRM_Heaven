# IQ_Transmitter (IQ Decoder/TX Utility)

`IQ_Transmitter` is a command-line decoder/transmitter utility for IQ files generated by the `MDI_to_IQ` project.

## What it does

- reads interleaved IQ input (`f32` or `s16`),
- optionally auto-loads and validates metadata from `<input.iq>.iqmeta`,
- performs nearest-point QPSK demapping,
- recovers payload bits,
- packs bits into bytes (MSB-first),
- writes recovered binary output,
- optionally streams IQ to FL2K through an external command,
- provides an M7 DRM TX pipeline (`--tx-drm`) with deterministic stage/logical-frame dumps, KPI validation, and compliance checks.

## Usage

```powershell
.\x64\Debug\IQ_Transmitter.exe <input.iq> <output.bin> [--format f32|s16] [--report <report.txt>] [--meta <file.iqmeta>] [--no-meta]
.\x64\Debug\IQ_Transmitter.exe <input.iq> --tx-fl2k --fl2k-cmd "<command reading raw u8 IQ on stdin>" [--format f32|s16] [--meta <file.iqmeta>] [--no-meta] [--tx-scale <v>] [--tx-headroom-db <db>] [--tx-clip] [--tx-shape] [--region <LAB|EU|US|HU>] --ota-lab-authorized
.\x64\Debug\IQ_Transmitter.exe <input.iq> --tx-drm [--format f32|s16] [--meta <file.iqmeta>] [--no-meta] [--dump-stages] [--dump-dir <path>] [--tx-scale <v>] [--tx-headroom-db <db>] [--tx-clip] [--tx-shape] [--m6-profile <robust|balanced|highrate>] [--kpi-out <kpi.txt>] [--run-id <id>] [--lab-tag <tag>] [--region <LAB|EU|US|HU>]
.\x64\Debug\IQ_Transmitter.exe --self-test
.\x64\Debug\IQ_Transmitter.exe --self-test-m2
.\x64\Debug\IQ_Transmitter.exe --self-test-m3
.\x64\Debug\IQ_Transmitter.exe --self-test-m4
.\x64\Debug\IQ_Transmitter.exe --self-test-m5
.\x64\Debug\IQ_Transmitter.exe --self-test-m6
.\x64\Debug\IQ_Transmitter.exe --self-test-m7
```

## FL2K Transmission Example

The app converts normalized IQ samples (`[-1, 1]`) to interleaved unsigned 8-bit samples and streams them to `--fl2k-cmd` via stdin.

```powershell
.\x64\Debug\IQ_Transmitter.exe .\mf_from_pcap.iq --tx-fl2k --fl2k-cmd "fl2k_file -s 12000000 -f u8 -c 2 -i -"
```

Use the exact `fl2k_file` arguments required by your local osmo-fl2k build.

## M7 DRM TX Pipeline Example

```powershell
.\x64\Debug\IQ_Transmitter.exe .\mf_from_pcap.iq --tx-drm --dump-stages --dump-dir .\m7_dumps --tx-scale 0.9 --tx-headroom-db 3 --tx-clip --tx-shape --m6-profile balanced --kpi-out .\m7_kpi.txt --run-id lab-001 --lab-tag room-a --region LAB
```

This mode includes M2 channel coding/interleaving, M3 FAC/SDC/MSC logical framing, M4 OFDM mapping, and M5 TX conditioning. M6 adds profile-based runs plus reproducible KPI exports. M7 adds compliance fields (`m7_occupied_bw_ratio`, `m7_spur_suppression_db`, `m7_region_allowed`, `m7_validation`, `m7_validation_reason`).
